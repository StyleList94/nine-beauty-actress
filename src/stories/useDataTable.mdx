# useDataTable

`@tanstack/react-table`을 래핑하여 정렬과 페이지네이션을 간편하게 설정하는 hook입니다.

## Parameter

- `data`: 테이블 데이터 배열 (필수) `TData[]`
- `columns`: 컬럼 정의 배열 (필수) `ColumnDef<TData, any>[]`
- `sorting`: 정렬 활성화. `true`면 클라이언트 정렬, `SortingState`면 초기 상태 지정 `boolean | SortingState`
- `onSortingChange`: 정렬 상태 변경 핸들러 (controlled mode) `OnChangeFn<SortingState>`
- `manualSorting`: 서버사이드 정렬 여부 `boolean`
- `pagination`: 페이지네이션 활성화. `true`면 기본 pageSize(10), 객체면 커스텀 pageSize `boolean | { pageSize: number }`
- `onPaginationChange`: 페이지네이션 상태 변경 핸들러 (controlled mode) `OnChangeFn<PaginationState>`
- `manualPagination`: 서버사이드 페이지네이션 여부 `boolean`
- `rowCount`: 서버사이드 페이지네이션 시 전체 행 수 `number`
- 그 외 `@tanstack/react-table`의 `TableOptions` passthrough 가능

## Return

- `Table<TData>` — tanstack 테이블 인스턴스. `DataTable` 컴포넌트의 `table` prop에 전달합니다.

## 특성

- 소비자가 `@tanstack/react-table`을 직접 import할 필요 없이 간편하게 사용할 수 있습니다
- 내부적으로 sorting/pagination 상태를 `useState`로 관리합니다
- `manualSorting`/`manualPagination` 옵션으로 서버사이드 모드를 지원합니다
- tanstack의 고급 옵션도 rest parameter로 passthrough 가능합니다

## 적용 요령

- `sorting: true`만 지정하면 클라이언트 정렬이 즉시 활성화됩니다
- 서버사이드 모드에서는 `manualSorting: true` + `onSortingChange`를 함께 사용합니다
- `pagination: { pageSize: 20 }`으로 페이지당 행 수를 커스텀할 수 있습니다
- 반환된 테이블 인스턴스는 반드시 `DataTable` 컴포넌트에 전달해야 합니다

## 예시

<Story of={useDataTableStories.Default} />

```tsx
import {
  useDataTable,
  DataTable,
  DataTableContent,
  DataTablePagination,
} from '@stylelist94/nine-beauty-actress';
import type { ColumnDef } from '@stylelist94/nine-beauty-actress';

const columns: ColumnDef<User>[] = [
  { accessorKey: 'name', header: '이름' },
  { accessorKey: 'email', header: '이메일' },
  { accessorKey: 'role', header: '역할' },
];

function UsersTable({ data }: { data: User[] }) {
  const table = useDataTable({
    data,
    columns,
    sorting: true,
    pagination: { pageSize: 20 },
  });

  return (
    <DataTable table={table}>
      <DataTableContent />
      <DataTablePagination />
    </DataTable>
  );
}
```
